# AgentMesh — 路演详细介绍

> 聊天优先的团队 Agent 协作平台：让每个团队拥有自己的 AI 工作大脑

---

## 一、项目定位与核心价值

### 1.1 一句话定位
**AgentMesh 是一个以聊天为核心入口的团队 AI Agent 协作平台**，让团队成员通过自然语言对话，即可调度多个专业 Agent 协同完成信息检索、数据分析、风险审查、经验沉淀等工作。

### 1.2 解决的痛点

| 痛点 | 现状 | AgentMesh 方案 |
|------|------|----------------|
| 团队经验流失 | 关键项目经验散落在文档、聊天、个人笔记中，新项目总从零开始 | 自动提取高价值经验，经审核后沉淀为团队记忆，新项目可即时检索复用 |
| 多源信息割裂 | 数据指标在 BI 平台、竞品资料在外部搜索、风险规则在法务文档 | 一个聊天窗口统一调度 data_agent、research_agent、risk_agent |
| 人工审批黑洞 | 高风险操作缺乏系统化审批流程，要么跳过审批要么全靠邮件 | 自动识别高风险操作，路由到收件箱等待人工确认，全程可审计 |
| AI 安全盲区 | 外部资料直接喂给 LLM，存在提示词注入风险 | 外部内容先过风险扫描，可疑内容隔离审核，不直接参与合成 |
| 单一 Agent 能力有限 | 通用 ChatBot 只能对话，无法执行研究、查数据、做审查 | 多 Agent 协作架构，个人 Agent 调度专业 Agent，任务在 BBS 看板流转 |

### 1.3 核心价值主张

- **效率价值**：将"查历史 → 找数据 → 看风险 → 写结论"从数小时压缩到数秒
- **安全价值**：AI 每一步操作有来源、有审核、有审计，企业级合规保障
- **知识价值**：团队经验自动沉淀、按需检索，组织记忆持续增值

### 2.2 核心设计理念

1. **聊天优先 (Chat-First)**：一切交互从聊天开始，用户通过显式 `$ skill` 调度资源
2. **多 Agent 协作 (Multi-Agent Collaboration)**：个人 Agent 作为调度中心，协调专业 Agent
3. **黑板共享 (Blackboard Pattern)**：Agent 之间通过 BBS 看板异步协作，而非紧耦合调用
4. **人工在环 (Human-in-the-Loop)**：高风险操作必须经过人工审批，AI 不越权
5. **边界清晰 (Integration Boundary)**：外部系统通过 Protocol 接口接入，核心不依赖具体实现

## 三、模块详解

---

### 模块一：智能聊天工作台

#### 功能描述
用户在聊天窗口中用自然语言提问或下达指令，若需要执行能力则通过显式 `$ skill` 进入工作流，系统完成：任务创建 → Agent 调度 → 证据收集 → LLM 合成 → 结果返回，整个流程在 1-2 秒内完成。

#### 核心能力
- **8 种显式 `$` skill**：查询记忆、生成 Brief、记录笔记、外部研究、数据查询、风险审查、沉淀经验、系统查询
- **显式工作流路由**：用户输入 `$` 后选择能力，避免隐式分类歧义
- **对话上下文保持**：线程级会话持久化，自动携带最近 10 条对话历史（4000 字符预算）
- **来源可追溯**：每条回答附带证据来源，不编造信息

#### 技术亮点
- **显式工作流路由**：规则只负责 `$` 命令解析，普通对话保持自然闲聊
- **多 API 风格兼容**：同时支持 OpenAI Chat Completions API 和 Responses API，适配不同模型服务
- **确定性兜底**：LLM 不可用时自动降级到本地规则回答，永不返回错误
- **Token 预算控制**：对话历史按条数 + 字符数双限制，避免 LLM 调用成本失控

#### 商业价值
- 降低团队使用 AI 的门槛，无需学习 Prompt 技巧
- 统一入口消除工具切换成本
- 对话即任务，结果即来源，零培训成本上手

---

### 模块二：多 Agent 协作引擎

#### 功能描述
以个人 Agent 为调度核心，协调 research_agent、data_agent、risk_agent 等专业 Agent，通过 BBS 看板异步协作完成任务。

#### Agent 角色

| Agent | 职责 | 触发意图 |
|-------|------|---------|
| Personal Agent | 工作流调度、任务创建、证据合成、回答生成 | 所有显式 skill |
| Research Agent | 外部资料检索（Web/Oxygen-CLI） | 查询记忆、外部研究、生成 Brief、沉淀经验 |
| Data Agent | 数据指标查询（本地/外部数据源） | 数据查询 |
| Risk Agent | 策略规则审查（提示注入/素材授权/高风险工具） | 风险审查 |

#### 核心能力
- **Agent 可配置**：每个用户拥有独立个人 Agent，可配置名称、状态、能力、模型、工具
- **工具显式授权**：工具按 Agent 授权，非全局暴露，安全可控
- **运行时状态追踪**：Agent 的在线状态、当前任务、最近活跃时间实时可见
- **多模型选择**：每个 Agent 可独立选择 LLM 模型，API Key 仅存服务端

#### 技术亮点
- **Acquisition Protocol**：外部获取通过 `AcquisitionAgent` Protocol 接口，Mock/Web/Oxygen/Composite 四种实现可插拔
- **CompositeAgent 聚合**：多获取源并行查询，结果自动合并去重，单源故障不影响其他
- **执行锁 (ExecutionLock)**：Agent 执行任务时获取排他锁，防止并发冲突
- **结构化交接 (StructuredHandoffPacket)**：Agent 间交接包含目标、当前结果、完成条件、阻塞点、协作需求

#### 商业价值
- 一个聊天入口 = 无限专业能力，按需扩展 Agent
- Agent 能力与权限解耦，安全与灵活并存
- 新 Agent 即插即用，无需修改核心逻辑

### 模块三：BBS 看板（Blackboard）

#### 功能描述
借鉴 AI 领域经典的黑板模型（Blackboard Architecture），为多 Agent 协作提供异步共享空间。Agent 在看板上发布请求、证据、风险、决策、交接记录，其他 Agent 读取后响应。

#### 核心能力
- **7 种帖子类型**：请求、证据、风险、摘要、决策、纠正、记忆候选
- **5 阶段协作流程**：讨论 → 执行 → 审查 → 阻塞 → 完成
- **执行锁机制**：Agent 获取排他锁后执行，释放后进入审查
- **结构化交接**：Agent 间交接包含目标、进度、阻塞点、协作需求
- **阅读标记**：帖子可标记已读，Agent 可知哪些信息已被消费
- **自动发布队列**：后台任务可排队到看板，手动或定时排空
- **权限过滤**：用户只能看到自己参与的或权限范围内的任务卡片

#### 技术亮点
- **Blackboard Pattern 实现**：松耦合的异步协作，Agent 无需直接调用彼此
- **后台 Worker**：可配置的自动发布 Worker，定时将队列中的帖子排空到看板
- **任务卡片聚合**：按任务维度聚合最新帖子、协作阶段、执行锁状态
- **线程级可见性**：基于聊天线程 + Agent 参与度双重过滤，确保数据隔离

#### 商业价值
- 团队可全局观察 Agent 工作进展，打破 AI 黑盒
- 协作过程可视化，问题可追溯
- 交接记录结构化，Agent 切换不丢失上下文

### 模块四：收件箱

#### 功能描述
个人 Agent 从 BBS 和项目动态中自动整理与用户相关的待处理事项，包括风险审核、工具审批、决策确认等，等待人工处理。

#### 核心能力
- **自动发现**：Agent 在工作过程中自动创建收件箱项，无需手动添加
- **三种处理方式**：确认（resolved）、稍后处理（snoozed，可设 TTL）、重新打开（open）
- **4 种事项类型**：工具调用审批、提示注入审核、决策评审、风险审查
- **状态追踪**：已确认时间、稍后处理时间、重新打开时间完整记录

#### 技术亮点
- **人在环中 (HITL) 闭环**：高风险操作必须经过人工确认后才执行，AI 不越权
- **Snooze TTL**：可设延迟提醒时间（最长 7 天），到期自动回到待处理
- **与 BBS 联动**：收件箱项与 BBS 帖子关联，点击可追溯完整上下文

#### 商业价值
- 合规场景的刚需：高风险操作必须有人签字
- 减少信息过载：只推送与你相关的，不是全局广播
- 审计友好：每个决策都有时间戳和操作人

---

### 模块五：记忆库

#### 功能描述
将团队经验从隐性知识转化为显性记忆，支持候选 → 审核 → 接受 → 沉淀的全生命周期管理。

#### 核心能力
- **6 种记忆状态**：草稿 → 提名 → 接受 → 争议 → 废弃 → 过期
- **4 层可见范围**：私有 → 项目 → 团队候选 → 团队已接受
- **来源追溯**：每条记忆附带证据来源，知其然更知其所以然
- **权限管控**：仅组长和管理员可将候选记忆接受为团队级
- **自动提名**：Agent 在对话中识别到高价值经验时，自动创建候选记忆

#### 技术亮点
- **Scope 升级机制**：候选记忆接受后自动从 `team_candidate` 升级到 `team_accepted`，权限随之变化
- **搜索权限过滤**：搜索结果按 personal/project/team 三级过滤，确保信息隔离
- **来源链**：记忆 → BBS 帖子 → 聊天消息 → 用户输入，完整追溯链

#### 商业价值
- 组织知识不再随人员流动而流失
- 新项目启动时可即时检索历史经验，避免重复踩坑
- 记忆需要审核才能沉淀，保证质量而非垃圾堆积

### 模块六：风险与审计

#### 功能描述
多层风险扫描引擎，覆盖提示注入、高风险工具、素材授权、人工确认四大类别，全程审计日志记录。

#### 4 大风险类别

| 类别 | 扫描内容 | 默认决策 |
|------|---------|---------|
| Prompt Injection | 外部内容中的"忽略之前指令"等注入信号 | needs_review |
| Tool Call | 批量抓取、内网访问、删除等高风险操作 | needs_review |
| Source Policy | 第三方素材、授权范围、使用期限 | needs_review |
| Approval | 人工确认、法务审批等正面信号 | allow |

#### 核心能力
- **三层风险扫描**：
  1. **外部内容扫描**：research_agent 返回的资料先过注入检测
  2. **工具请求扫描**：用户请求先检查是否涉及高风险操作
  3. **策略规则扫描**：基于可配置的持久化规则库进行综合评估
- **可配置策略规则**：管理员可在成员页面管理风险规则，无需改代码
- **三档决策**：allow（放行）→ needs_review（审核）→ block（阻断）
- **隔离机制**：可疑内容标记为 `needs_review`，暂不参与 LLM 合成

#### 技术亮点
- **规则引擎 + 策略持久化**：默认规则保证零配置安全，持久化规则支持业务定制
- **内容隔离**：可疑内容不直接喂给 LLM，而是存入审计库 + 路由到收件箱
- **完整审计链**：每个操作（创建任务、发布帖子、审批工具、接受记忆等）都有 AuditEvent 记录

#### 商业价值
- 企业级 AI 安全基座：提示注入防护是 AI 应用的刚需
- 合规审计一键可查：谁在什么时间做了什么决策，全程留痕
- 策略可配置：不同行业/场景的风险规则可由管理员定制，无需开发介入

### 模块七：搜索

#### 功能描述
跨聊天消息、活动记录、BBS 证据、记忆项的统一搜索，按工作空间、项目和权限范围过滤。

#### 核心能力
- **多源统一搜索**：一个查询同时搜索消息、活动、帖子、记忆
- **权限感知**：按 personal/project/team 三级可见范围过滤结果
- **上下文过滤**：搜索限定在当前工作空间和项目范围内
- **来源关联**：搜索结果附带来源信息，可追溯

#### 技术亮点
- **SQLite 全文检索**：基于 JSON payload 的全量扫描，后续可无缝升级到 FTS5
- **Scope 过滤**：搜索结果在 API 层按用户权限过滤，确保不越权

#### 商业价值
- 历史经验一键搜索，避免重复劳动
- 权限隔离确保信息安全

---

### 模块八：数据连接器

#### 功能描述
可扩展的数据源连接器注册表，支持本地指标查询和外部数据源（Oxygen-CLI）接入。

#### 核心能力
- **连接器注册表**：`DataSourceRegistry` 路由查询到已注册的连接器
- **本地指标连接器**：`LocalMetricsConnector` 提供模拟指标数据
- **Oxygen-CLI 数据连接器**：支持 metasearch、oxygen-comment、bdp-copilot 等 CLI 工具
- **只读安全边界**：Oxygen 数据连接器只允许 read-only 操作

#### 技术亮点
- **Protocol 接口**：`DataSourceConnector` Protocol 定义标准接口，外部实现即插即用
- **智能指标提取**：从用户输入中自动识别指标类型（CTR、入口效率、转化率）
- **多 CLI 适配**：metasearch、o2-kb、oxygen-comment、bdp-copilot 等不同 CLI 协议统一适配

#### 商业价值
- 企业数据资产不锁死在单一平台，按需接入
- 只读安全边界防止 AI 误操作写入生产数据

---

### 模块九：文档解析

#### 功能描述
用户上传文档，系统解析后作为来源存储，供 Agent 检索引用。

#### 核心能力
- **多格式支持**：TXT、Markdown、PDF
- **组合解析器**：`CompositeDocumentParser` 按文件类型自动路由到对应解析器
- **PDF 解析**：基于 PyMuPDF 的 PDF 解析器，提取文本和页数
- **来源关联**：解析后的文档自动创建 Source 记录，可被 BBS 帖子和记忆引用

#### 技术亮点
- **Protocol 边界**：`DocumentParser` Protocol 接口，外部解析器（Word、PPT 等）可即插即用
- **标题智能提取**：Markdown 从 `# 标题` 提取，PDF 从首行文本提取
- **懒加载**：PDF 解析器仅在安装了 PyMuPDF 时启用，不影响核心功能

#### 商业价值
- 将静态文档转化为可检索的活知识
- 上传即入库，无需手动整理

### 模块十：外部集成

与外部系统的集成边界，包括 Web 研究提供商、Oxygen-CLI 内部能力、LLM 模型服务。

| Provider | 命令 | 适用场景 |

#### Oxygen-CLI 集成

| CLI | 能力 | 触发方式 |
|-----|------|---------|
| metasearch | 内部知识搜索 | research / data 查询 |
| o2-kb | 知识库召回 | research 查询 |
| oxygen-comment | 评论数据查询 | data 查询 |

#### 技术亮点

#### 商业价值
- 不绑定任何特定供应商，企业可按需接入自有系统
- Oxygen-CLI 作为内部能力接入层，兼顾内网安全与外部灵活性

### 模块十一：认证与权限

#### 功能描述
基于 Cookie 会话的轻量认证体系，支持三级角色权限控制。

#### 角色权限矩阵

| 能力 | 普通用户 | 组长 | 管理员 |
|------|---------|------|--------|
| 管理自己的个人 Agent | ✅ | ✅ | ✅ |
| 查看公共 Agent | ✅ | ✅ | ✅ |
| 接受候选团队记忆 | ❌ | ✅ | ✅ |
| 管理公共 Agent | ❌ | ❌ | ✅ |
| 创建用户/重置密码 | ❌ | ❌ | ✅ |
| 创建工作空间/项目 | ❌ | ❌ | ✅ |
| 管理风险策略规则 | ❌ | ❌ | ✅ |
| 同步 Oxygen 工具 | ❌ | ❌ | ✅ |

#### 技术亮点
- **PBKDF2-SHA256** 密码哈希，120,000 次迭代
- **HttpOnly Cookie** + Bearer Token 双模式认证
- **会话管理**：12 小时 TTL，支持手动撤销
- **密码修改自动撤销旧会话**：修改密码后所有设备自动登出

#### 商业价值
- 开箱即用的认证方案，MVP 阶段无需对接 SSO
- 角色权限灵活，适配不同组织结构
- 后续可无缝升级到 OAuth/SSO

## 四、技术亮点总结

### 4.1 架构级亮点

| # | 亮点 | 说明 |
|---|------|------|
| 1 | **Chat-First 架构** | 一切从对话开始，显式 `$` skill → 任务自动创建 → Agent 自动调度，用户零学习成本 |
| 2 | **Blackboard Pattern** | 多 Agent 通过 BBS 看板松耦合协作，而非紧耦合调用，Agent 可独立开发/部署/升级 |
| 3 | **Protocol 边界** | 所有外部依赖（搜索、数据、文档、LLM）通过 Protocol 接口隔离，核心零依赖 |
| 4 | **Human-in-the-Loop** | 高风险操作必须人工审批，AI 不越权，满足企业合规要求 |
| 5 | **显式工作流路由** | `$` 命令解析（毫秒级、零成本）+ LLM 生成（高准确率），兼顾效率与质量 |

### 4.2 安全级亮点

| # | 亮点 | 说明 |
|---|------|------|
| 1 | **提示注入防护** | 外部内容过注入检测，可疑内容隔离不参与 LLM 合成 |
| 2 | **高风险工具审批** | 批量抓取、内网访问等操作路由到收件箱等待审批 |
| 3 | **内容隔离** | 可疑证据标记为 `needs_review`，不入 LLM 上下文 |
| 4 | **完整审计链** | 每个操作生成 AuditEvent，全链路可追溯 |
| 5 | **工具显式授权** | 工具按 Agent 授权，非全局暴露，最小权限原则 |

### 4.3 工程级亮点

| # | 亮点 | 说明 |
|---|------|------|
| 1 | **单文件前端** | 全部 UI 在一个 HTML 文件中，无构建工具，零部署复杂度 |
| 2 | **SQLite 泛化存储** | 单表 JSON payload 设计，新增模型类型无需 DDL 变更 |
| 3 | **多 API 风格兼容** | 同时支持 Chat Completions 和 Responses API |
| 4 | **确定性兜底** | LLM 不可用时自动降级到本地规则回答，永不 500 |
| 5 | **环境变量驱动** | 所有外部集成通过环境变量配置，零代码切换 |

## 五、典型使用场景

### 场景 1：新项目启动
> 产品经理："去年 618 家电会场做过什么方案？效果如何？"

1. 用户输入 `$research.request`
2. Research Agent 在团队记忆和外部资料中检索
3. 返回："2025 年 618 家电会场尝试沉浸式头图，首屏入口点击下降，后续改为效率型楼层结构"
4. 附带来源："2025 618 家电会场复盘"

**价值**：从"翻文档找半天"到"3 秒拿到结论 + 来源"

### 场景 2：数据指标查询
> 运营："这个活动的点击率怎么样？"

1. 用户输入 `$data.query`
2. Data Agent 通过连接器查询指标数据
3. 返回："CTR 为 12.3%，来源：local_metrics"

**价值**：从"打开 BI 平台找指标"到"一句话查数据"

### 场景 3：风险审查
> 设计师："这批素材可以用吗？有没有授权问题？"

1. 用户输入 `$risk.review`
2. Risk Agent 根据策略规则包扫描
3. 返回："发现风险确认项：第三方素材需确认授权范围"
4. 自动创建收件箱项等待确认

**价值**：从"法务邮件来回 3 天"到"实时风险提示 + 审批流程"

### 场景 4：经验沉淀
> 团队负责人："把这次大促的经验沉淀下来"

1. 用户输入 `$memory.propose`
2. 自动从对话和证据中提取高价值结论
3. 创建候选记忆："大促会场首屏结构偏好：转化优先时保证核心入口密度"
4. 组长在记忆库审核后接受为团队记忆

**价值**：从"经验随人走"到"知识自动沉淀、组织持续增值"

## 九、总结

**AgentMesh 不是一个 ChatBot，而是一个团队 AI 协作操作系统：**

- **入口层**：聊天优先，显式 `$` skill，零学习成本
- **协作层**：多 Agent 通过 BBS 看板松耦合协作，可扩展可替换
- **安全层**：提示注入防护 + 高风险审批 + 策略规则 + 全链路审计
- **知识层**：经验自动沉淀，按需检索，组织记忆持续增值
- **集成层**：Protocol 边界，外部系统即插即用，不锁定供应商

**一句话：让每个团队拥有一个安全、可审计、会协作的 AI 工作大脑。**
